# Web Proje Öğrenme Sayfası

Bu sayfa, GitHub ve GitHub Pages öğrenmek için hazırlanmış küçük bir uygulama alanıdır. Her düzenlemede dosya değiştirmeyi, commit atmayı ve yayınlanan sayfayı kontrol etmeyi öğreneceğiz.

## Bugünkü hedefimiz

- GitHub deposunun dosya yapısını tanımak
- `docs/index.md` dosyasını düzenlemek
- Değişikliği commit olarak kaydetmek
- GitHub Pages mantığını anlamak

## Bu projede neler var?

Bu depo bir **öğretici paket** şablonu olarak hazırlanmıştır. İçinde Python paketi, örnekler ve MkDocs ile oluşturulan dokümantasyon sayfaları bulunur.

| Bölüm | Açıklama |
|---|---|
| `README.md` | Projenin genel tanıtımı |
| `docs/` | Web sayfasında yayınlanan dokümanlar |
| `mkdocs.yml` | Site ayarları ve menü yapısı |
| `src/` | Python paket kodları |
| `.github/workflows/` | Otomatik yayın ve iş akışı dosyaları |

## GitHub öğrenme akışı

1. Dosyayı aç.
2. Küçük bir değişiklik yap.
3. Commit mesajı yaz.
4. Değişikliği GitHub'a gönder.
5. Yayınlanan sayfayı kontrol et.

## İlk kavramlar

### Repository

Repository, projenin GitHub üzerindeki klasörüdür. Kodlar, dokümanlar ve ayarlar burada tutulur.

### Commit

Commit, yapılan değişikliğin kayıt noktasıdır. Her commit kısa ve açıklayıcı bir mesajla kaydedilir.

### Branch

Branch, ana projeyi bozmadan deneme yapmak için kullanılan çalışma dalıdır.

### Pull Request

Pull Request, yapılan değişikliklerin ana dala eklenmeden önce incelenmesini sağlar.

## Sonraki adım

Bir sonraki değişiklikte `installation.md` ve `usage.md` dosyalarını daha düzenli hale getirerek GitHub Pages menüsünü birlikte geliştireceğiz.
